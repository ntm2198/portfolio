Technical Details

The projection pipeline started as a conventional MadMapper setup, with the laptop routing mapped output to the projector and treating a 3D arm model as the target surface. When the modeled arm did not behave predictably enough for the live demo, the workflow shifted from object-level alignment to UV-space control in Blender.

  1. Input + control MadMapper

    Managed the projection scene, output routing, and surface-mapping controls.

  2. Render host Laptop

    Ran the mapping stack and sent the composed visual output to the display chain.

  3. Display output Projector

    Projected the mapped feed onto the participant-facing physical surface.

  4. Initial geometry 3D arm model failed

    The model-to-projector correspondence was too unstable for accurate alignment.

  5. Fallback mapping Blender UV layout

    Converted the problem into UV coordinates so artwork could be placed on mesh islands.

  6. Live texture feed Spout Browser

    Streamed drawings from an open-source web drawing app into the projection workflow.

  7. Calibration pass Manual UV mesh adjustment

    Tuned the UV islands by hand until the projected tattoo marks landed correctly.

Why the pivot mattered

The final system treated the drawing app as a live texture source and the UV mesh as the correction layer between screen-space artwork and the projected arm surface. That made the prototype less dependent on a perfect 3D model and more resilient to the real alignment constraints of a public exhibit.
